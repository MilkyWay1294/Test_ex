# Task Manager API

REST API сервис для командного управления задачами с ролевой моделью доступа, историей изменений (аудитом), комментариями, кэшированием в Redis и аналитикой.

## Технологический стек
- **Язык**: Go (Gin)
- **База данных**: MySQL 8.0
- **Кэширование**: Redis 7
- **Контейнеризация**: Docker & Docker Compose

---

## Архитектура и возможности

### 1. Ролевая модель
Поддерживаются 3 роли участников команд:
- `owner` (создатель команды): имеет полные права, может приглашать участников.
- `admin`: может приглашать участников, управлять задачами.
- `member`: может просматривать задачи и обновлять их статусы/оставлять комментарии.

Для оптимизации проверки прав доступа роли пользователей в командах кэшируются в Redis. При изменении роли кэш инвалидируется.

### 2. Аудит изменений (История задач)
Все изменения полей задач (`title`, `description`, `status`, `assignee_id`) отслеживаются и записываются в таблицу `task_history` в рамках одной транзакции СУБД.

### 3. Кэширование
- Проверка ролей пользователя в командах (Redis).
- Детали задач (кэшируются в Redis на 1 час, инвалидируются при любом изменении задачи).
- Сводная аналитика команды (кэшируется на 1 минуту для защиты от высокой нагрузки).

### 4. Сложная аналитика (Сложные SQL-запросы)
Эндпоинт аналитики команды агрегирует данные из нескольких таблиц:
- Общее количество задач команды.
- Количество задач в разрезе статусов (`todo`, `in_progress`, `review`, `done`).
- Индивидуальная статистика продуктивности участников (активные задачи, выполненные задачи) через сложные `LEFT JOIN` с группировкой по исполнителям.
- Лента последних 10 действий в команде на основе логов истории изменений.

## Запуск проекта

Убедитесь, что у вас установлен Docker и запущен Docker Desktop.

1. **Скомпилируйте бинарный файл под Linux**:
   ```bash
   CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o main_linux ./src/main.go
   ```

2. **Запустите контейнеры**:
   ```bash
   docker-compose up --build -d
   ```

Сервер будет доступен по адресу `http://localhost:8080`.

## API Эндпоинты

### Аутентификация
- `POST /api/v1/register` — регистрация (`username`, `email`, `password`).
- `POST /api/v1/login` — авторизация, возвращает JWT токен.

### Команды (требуют авторизации)
- `POST /api/v1/teams` — создание команды.
- `GET /api/v1/teams` — список команд, в которых состоит пользователь.
- `POST /api/v1/teams/:id/invite` — приглашение пользователя в команду (только для `owner`/`admin`).
- `GET /api/v1/teams/:id/analytics` — сводная аналитика по команде.

### Задачи (требуют авторизации)
- `POST /api/v1/teams/:id/tasks` — создание задачи в команде.
- `GET /api/v1/teams/:id/tasks` — список задач команды (с фильтрацией по `status` и `assignee_id`).
- `GET /api/v1/tasks/:id` — детальная информация о задаче (кэшируется в Redis).
- `PATCH /api/v1/tasks/:id` — редактирование задачи/смена статуса (записывает изменения в историю).
- `GET /api/v1/tasks/:id/history` — история изменений задачи (аудит).

### Комментарии (требуют авторизации)
- `POST /api/v1/tasks/:id/comments` — добавить комментарий.
- `GET /api/v1/tasks/:id/comments` — получить список комментариев к задаче.
